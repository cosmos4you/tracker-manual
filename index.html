Регламент работы в Яндекс Трекере

Как пользоваться Яндекс Трекером · Ромашка РА

1Общие положения

  • Яндекс Трекер — единая система учёта задач агентства. Адрес: tracker.yandex.ru, вход под рабочим Яндекс-аккаунтом. Доступно мобильное приложение «Яндекс Трекер».
  • Все рабочие задачи фиксируются в Трекере. Договорённости в чатах и на встречах оформляются задачей в тот же день.
  • Каждая задача имеет дедлайн и тег клиента.
  • Статусы задач поддерживаются в актуальном состоянии исполнителем.

2Очереди и зоны ответственности

Очередь — раздел направления. Задача ставится в очередь по сути работы:

ОчередьКлючЧто входитОтветственный
SMMSMMSMM-проекты, брифы, сметы, презентации и кейсы, event, подключение дизайнеров и копирайтеровАлина
ПродакшнPRODСъёмки, монтаж, дизайн-продакшен, производство контентаАлина
ORMORMУправление репутацией, отзывы, SERMНастя Филиппова
PRPRPR-активности, СМИ, публикацииНастя Филиппова
ИнфлюенсINFLИнфлюенс-маркетинг, работа с блогерами, интеграцииНастя Филиппова
  • Если исполнитель не указан, задача автоматически назначается ответственному очереди, который распределяет её внутри команды.
  • Задача, поставленная не в ту очередь, переносится без потерь: меню «…» в карточке → «Переместить в другую очередь».

3Постановка задачи

  1. Создание. Кнопка ➕ в левом меню → «Задачу».
  2. Очередь. Выбрать по таблице раздела 2.
  3. Название. Сначала клиент (бренд), затем суть задачи глаголом: «Клиент Х — подготовить смету SMM», «Бренд Y — согласовать подборку блогеров». Формулировки «Смета», «Срочно» не допускаются.
  4. Описание. Заполняется так, чтобы исполнитель не задавал уточняющих вопросов:
    • контекст — клиент, проект, зачем нужна работа;
    • что сделать — конкретное действие и объём;
    • результат — что считается выполненным (файл, публикация, согласованный документ);
    • материалы — ссылки на бриф, диск, референсы, доступы.
  5. Дедлайн. Указывается всегда. У длинных задач — промежуточные контрольные точки в описании.
  6. Приоритет. По умолчанию «Средний». «Высокий»/«Критичный» — только для задач, которые блокируют другую работу.
  7. Теги. Тег клиента обязателен, например клиент-х. Единый тег на всех задачах клиента позволяет одним фильтром получить всю работу по нему во всех очередях.
  8. Исполнитель. Указывается, если известен. Если нет — поле остаётся пустым, задача уходит ответственному очереди.
  9. Наблюдатели. Добавляются все, кто должен следить за ходом работы (в том числе автор, если ставит задачу другому).
  10. Проект. Если задача — часть комплексного запуска, она привязывается к проекту в поле «Проект» (структура проектов — раздел 4).
  11. Связи и подзадачи. В карточке задачи: «Связать» — указать зависимость («связана с», «блокирует», «является подзадачей»); большие задачи разбиваются на подзадачи или чек-лист.
Образец описания: «Клиент Х, запуск линейки в сентябре. Подобрать 15–20 бьюти-блогеров 50–150К подписчиков, ER от 3%, без интеграций с конкурентами. Результат — таблица: ник, охваты, цена, контакт. Бриф: [ссылка]. Дедлайн 10.07, черновик — 7.07».

4Структура работ: задачи, проекты, портфели

Инструмент выбирается по масштабу работы:

МасштабИнструмент
Отдельная работаЗадача в очереди с тегом клиента — основной формат
Работа с этапами (например, организация съёмки)Задача с подзадачами или чек-листом
Комплексный запуск на несколько направлений (SMM + продакшн + инфлюенс)Проект: создаётся через ➕ → «Проект», к нему привязываются задачи из разных очередей. У проекта свой дедлайн, ответственный и прогресс выполнения
Крупный клиент с несколькими проектами (запуск, сопровождение, event)Портфель — объединяет проекты клиента. Раздел «Проекты и портфели» в левом меню
  • Тег клиента проставляется на задачах на любом уровне структуры.
  • Переход на уровень выше выполняется только при реальной необходимости: если работа укладывается в задачу — проект не создаётся.

5Статусы и порядок работы с задачей

Открыт→ В работе→ Закрыт

  • Исполнитель переводит задачу «В работе», приступая к ней, и «Закрыт» — после передачи результата. Статус меняется кнопкой в карточке задачи.
  • Обсуждение ведётся в комментариях задачи, а не в личных чатах. Упоминание @имя отправляет уведомление коллеге.
  • При риске срыва дедлайна исполнитель сообщает об этом в комментарии заранее и предлагает новый срок.
  • Файлы прикладываются в задачу; объёмные исходники — на Диск со ссылкой в задаче.
  • Закрытую задачу можно вернуть в работу сменой статуса.

6Контроль и уведомления

  • «Мои задачи» (значок ☑ в левом меню) — все задачи сотрудника: как исполнителя, автора и наблюдателя. Проверяется ежедневно.
  • Почтовые уведомления включены по умолчанию: назначение задачи, комментарии, упоминания, смена статуса. Проверка: шестерёнка внизу левого меню → Настройки → Уведомления → канал «Почта».
  • Руководителям очередей (Алина, Настя): в настройках своей очереди → «Уведомления» включить подписку на событие «Задача создана» — для контроля всех новых задач направления.
  • Доски — канбан-представление задач по колонкам статусов; используются на планёрках. Создание: ➕ → «Доску».
  • Фильтры — список задач фильтруется по исполнителю, статусу, дедлайну, тегу; фильтр можно сохранить.
